## What It Does

Discord servers accumulate channels the same way inboxes accumulate emails: fast, chaotic, and quietly. This tool runs a weekly sweep of your server and reports back with organic intelligence:

- **HOT / WARM / COOL / DEAD scoring** for every channel based on message recency and frequency
- **Topic inference** — keyword-based detection of what's actually being discussed vs. what a channel was named for
- **Archive candidates** — dead channels (30+ days silent) listed for cleanup
- **Delta vs. last week** — tracks which channels are heating up or cooling down since the previous run
- **Category health** — flags categories where 70%+ of channels are dead
- **AI-generated infographic** — a visual summary image posted directly to your report channel via [nano-banana-pro](https://github.com/your-org/nano-banana-pro)

The report is posted as a Discord embed with a bot mention so you never have to go looking for it.

## Setup

### 1. Clone the repo

```bash
git clone https://github.com/BusyBee3333/discord-server-intelligence.git
cd discord-server-intelligence
```

### 2. Configure

```bash
bash scripts/setup.sh
```

This copies `config.example.json` → `config.json` and creates the `output/` directory. Then edit `config.json`:

```json
{
  "discord": {
    "botToken": "YOUR_DISCORD_BOT_TOKEN",
    "guildId": "YOUR_GUILD_ID",
    "reportChannelId": "YOUR_CHANNEL_ID"
  },
  "gemini": {
    "apiKey": "YOUR_GEMINI_API_KEY"
  },
  "analysis": {
    "hotThresholdDays": 2,
    "hotThresholdMessages": 5,
    "deadThresholdDays": 30,
    "lookbackMessages": 50
  }
}
```

**Required values:**
| Key | Where to get it |
|---|---|
| `discord.botToken` | [Discord Developer Portal](https://discord.com/developers/applications) → Your Bot → Token |
| `discord.guildId` | Server Settings → Widget → Server ID (enable Developer Mode first) |
| `discord.reportChannelId` | Right-click channel → Copy Channel ID |
| `gemini.apiKey` | [Google AI Studio](https://aistudio.google.com/app/apikey) |

### 3. Install dependencies

```bash
npm install
```

*(No external npm dependencies — pure Node.js fetch. This step is a no-op but good practice.)*

## Usage

### Run analysis only (posts embed to Discord)

```bash
npm run analyze
```

### Generate AI infographic only (reads from last analysis)

```bash
npm run report
```

### Full pipeline (analyze → generate → post)

```bash
npm run full
```

## Automating with Cron (weekly Sunday 8pm)

```bash
crontab -e
```

Add:

```cron
0 20 * * 0 cd /path/to/discord-server-intelligence && npm run full >> output/cron.log 2>&1
```

Or use a `.env`-style approach if your bot token lives in the environment:

```cron
0 20 * * 0 cd /path/to/discord-server-intelligence && DISCORD_BOT_TOKEN=your_token npm run full >> output/cron.log 2>&1
```

## What the Report Includes

Each weekly report includes:

| Section | Description |
|---|---|
| **Activity Overview** | Total counts of HOT / WARM / COOL / DEAD channels |
| **Most Active This Week** | Top channels by message count with inferred topics |
| **Archive Candidates** | Channels silent for 30+ days with last-seen timestamp |
| **Trending Up** | Channels that jumped from COOL/WARM → HOT vs. last week |
| **Category Health** | Flags categories where most channels are dead |
| **AI Infographic** | Visual summary image generated by nano-banana-pro |

### Activity Scoring

| Score | Criteria |
|---|---|
| 🔥 **HOT** | Last message ≤ 2 days ago AND ≥ 5 messages in the past 7 days |
| 🌡️ **WARM** | Last message ≤ 7 days ago AND ≥ 2 messages |
| 🧊 **COOL** | Last message ≤ 30 days ago |
| 💀 **DEAD** | Last message > 30 days ago (or never) |

### Topic Inference

Keyword-based topic extraction identifies what's actually being discussed:

- `upwork`, `gig`, `client`, `proposal` → **Upwork/freelancing**
- `github`, `repo`, `pr`, `commit` → **GitHub/code**
- `agent`, `bot` → **Agent/bot work**
- `discord`, `channel`, `server` → **Server organization**
- *...and more (see `src/utils/topics.mjs`)*

## State & Delta

After each run, `output/state.json` is saved with channel activity snapshots. The next run loads this file to compute deltas — which channels heated up or cooled down since last week.

The `output/` directory is gitignored so your analysis history stays local.
